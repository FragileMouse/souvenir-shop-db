## 🗄️ Схема базы данных
 
| Таблица | Описание |
|---|---|
| `category` | Категории товаров |
| `product` | Каталог товаров с ценами и артикулами |
| `client` | Клиентская база (имя, город, дата рождения) |
| `purchase` | Заказы (дата, клиент) |
| `purchase_details` | Состав заказа (товар, количество) |
 
СУБД: **PostgreSQL**
 


## 📊 Реализованные запросы
 
| № | Задача | Использованные техники |
|---|---|---|
| 1 | Количество позиций в категории «Брелки» | `INNER JOIN`, `WHERE` |
| 2 | Выручка за февраль 2026 | подзапрос, `SUM` |
| 3 | Самый популярный товар за всё время | `CTE`, `MAX` |
| 4 | Города клиентов по убыванию заказов | `GROUP BY`, `ORDER BY` |
| 5 | Любимый товар каждого клиента | `CTE`, оконная функция `MAX OVER PARTITION BY` |
| 6 | Клиенты без ни одного заказа | `NOT IN`, подзапрос |
| 7 | Средний возраст покупателей | `AGE`, `AVG`, `EXTRACT` |
| 8 | Самый дорогой товар в каждой категории | `GROUP BY`, `MAX` |
| 9 | Выручка и количество заказов по месяцам (янв–март 2026) | `CASE`, `SUM`, `COUNT` |
| 10 | Категории с наибольшим числом непроданных товаров | `CTE`, `NOT IN`, `COUNT` |
 

## 💡 Выводы по данным
 
- Основная масса клиентов — из **Санкт-Петербурга** (42 заказа), где расположены физические точки магазина. Казань и Москва делят второе место — потенциал для локальной рекламы или открытия новой точки.
- Самый продаваемый товар — **Открытка «фитнес»**.
- Наибольшее число **ни разу не проданных позиций** — в категории «Открытки» (19 позиций). Стоит пересмотреть ассортимент.
- В ходе работы с реальными данными магазина выявлены проблемы: дублирующиеся артикулы, разнобой в кавычках, отсутствующие артикулы в разделе «Канцелярия».
